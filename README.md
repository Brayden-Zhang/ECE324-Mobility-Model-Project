# Route Rangers

This repository is organized with the CCDS (`Cookiecutter Data Science`) structure so the project has a clean separation between data, code, notebooks, models, and reports from the start.

## Quick start

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -e .
python -m pip install -r requirements.txt
```

Useful commands:

```bash
make create_environment
make install
make test
```

## Project layout

```text
.
├── .github/workflows/ci.yml   # Basic GitHub Actions test workflow
├── data/                      # Raw, external, interim, and processed datasets
├── docs/                      # Longer-form documentation
├── models/                    # Trained model artifacts (gitignored)
├── notebooks/                 # Exploratory and experiment notebooks
├── references/                # Papers, specs, and background material
├── reports/                   # Generated analyses and exported figures
├── src/route_rangers/         # Reusable project code
└── tests/                     # Basic automated checks
```

## CCDS conventions used here

- `data/raw/` stores immutable source data.
- `data/external/` stores third-party inputs that are not generated by this repo.
- `data/interim/` stores cleaned but intermediate datasets.
- `data/processed/` stores model-ready datasets.
- `src/route_rangers/` stores reusable code instead of notebook-only logic.
- `reports/figures/` stores exported charts for presentations or writeups.

## Suggested workflow

1. Drop source files into `data/raw/` or `data/external/`.
2. Implement ingestion and cleaning in `src/route_rangers/data/`.
3. Build feature pipelines in `src/route_rangers/features/`.
4. Train and evaluate models from `src/route_rangers/models/`.
5. Keep experiments in `notebooks/` and move stable logic into `src/`.
